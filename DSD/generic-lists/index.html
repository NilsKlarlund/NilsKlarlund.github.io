Generic list example

Hugh S. Field-Richards (hsfr@hydra.dra.hmg.gb) kindly provided the following example as a challenge to a schema language:
  • model a generic ordered list as an element oList that contains a sequence of listItem elements, and
  • allow the description of useages of such ordered lists, where a usage is determined by the allowed content an listItem element in a oList element has in a specific context.
As an example, we find below two different usages of oList, namely
  • inside a forenames element, list items contain a name element, and
  • inside a webbs element, list items contain a qualification element: 
 <?dsd URI="persons.dsd"?>
 <person>
  <surname>Klarlund</surname>
  <forenames>
    <oList>
      <listItem><name>Nils</name></listItem>
    </oList>
  </forenames>
  <webbs>
    <oList>
      <listItem><qualification>C++</qualification></listItem>
      <listItem><qualification>XML</qualification></listItem>
    </oList>
  </webbs>
</person>
This situation can easily be handled using DSDs. We define general ordered lists in generic-list.dsd (pretty-printed) (generic-list.dsd (dsd)); in this DSD, the ordered list item element is specified as empty, but a constraint listItem-constraint is defined for amending the definition. And, the person elements and their usages of ordered lists are then explained by context-sensitive additions to the meaning of a listItem-constraint generic-list.dsd (pretty-printed) (persons.dsd (dsd))
